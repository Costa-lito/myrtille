\section{Introduction}
Myrtille est une charte graphique pensée pour produire des documents techniques avec un rendu clair et moderne. Son utilisation est faite pour être simple grâce à l'utilisation d'une classe \LaTeX{} standard.

% ── SECTION 4 : CRÉATION DE BADGES PERSONNALISÉS ───────────────
\section{Création de Badges Personnalisés}

La classe met à disposition une macro racine \ci{\textbackslash badge[couleur]\{texte\}} qui permet de générer de nouveaux badges en quelques secondes.

\subsection{À partir d'une couleur existante}
Si la couleur dont vous avez besoin figure déjà dans la palette (par exemple \texttt{infoGreen}), ajoutez simplement une nouvelle commande dans le préambule de votre document :

\begin{texblock}
\newcommand{\badgeNouveau}{\badge[infoGreen]{Nouveau}}
\end{texblock}

\textbf{Résultat visuel :} \badgeNouveau

\subsection{À partir d'une couleur inédite}
Bien que ce ne soit pas recommandé, si vous souhaitez utiliser une teinte qui n'est pas dans la charte standard, vous devez la définir avec \ci{\textbackslash definecolor}, puis créer le badge :

\begin{texblock}
% 1. Déclarer la couleur dans le préambule
\definecolor{customPurple}{HTML}{8E44AD}

% 2. Créer la commande associée
\newcommand{\badgePerso}{\badge[customPurple]{Spécial}}
\end{texblock}

\textbf{Résultat visuel :} \badgePerso

\subsection{Boîte d'Information (\texttt{infobox})}
\begin{infobox}
    \textbf{Astuce :} Ceci est une boîte d'information générée avec \ci{\textbackslash begin\{infobox\}}. Elle utilise les couleurs \texttt{accentPale} pour le fond et \texttt{accent} pour la bordure. Elle est idéale pour les remarques, astuces ou compléments d'information.

\subsection{Boîte d'Avertissement (\texttt{warnbox})}
\begin{warnbox}
    \textbf{Attention :} Ceci est une boîte d'avertissement générée avec \ci{\textbackslash begin\{warnbox\}}. Elle utilise des nuances de jaune/brun pour capter l'attention de l'utilisateur sur une action potentiellement destructrice ou une contrainte forte.
\end{warnbox}

\subsection{Note de Marge (\texttt{sidenote})}
\sidenote{Ceci est une note latérale (\texttt{\textbackslash sidenote}). Elle utilise un fond discret \texttt{sidebarBg} et une bordure \texttt{border}. Parfaite pour les métadonnées, les renvois bibliographiques ou des détails techniques très spécifiques.}

\subsection{Notes de bas de page (Symboliques)}

La classe Myrtille utilise par défaut un système de notes de bas de page symboliques\footnote{Ceci est un exemple de note de bas de page symbolique.} plutôt que numériques. Cela permet d'éviter toute confusion avec des appels de bibliographie ou des exposants mathématiques, tout en apportant une touche d'élégance éditoriale.

L'ordre d'apparition des symboles (géré automatiquement par \LaTeX) est le suivant :

\begin{enumerate}
    \item Astérisque (*)
    \item Obèle / Dague (\textdagger)
    \item Double obèle (\textdaggerdbl)
    \item Section (\textsection)
    \item Pied de mouche (\textparagraph)
    \item Parallèles (\textbardbl)
\end{enumerate}

\textbf{Note :} Si le document comporte plus de 6 notes sur une même page, le système doublera automatiquement les symboles (**, \textdagger\textdagger, etc.).

% ── SECTION 6 : ENVIRONNEMENTS MATHÉMATIQUES ───────────────────
\section{Environnements Mathématiques}

La classe Myrtille intègre des boîtes spécifiques pour mettre en valeur les théorèmes, propriétés et définitions. Ces environnements sont numérotés automatiquement par section et supportent les références croisées.

\subsection{Théorèmes et Propriétés (\texttt{myrthm})}

L'environnement \ci{myrthm} utilise la couleur bleue principale (\texttt{accent}) pour encadrer les résultats mathématiques ou logiques importants. Il prend deux arguments obligatoires : le titre du théorème, et un label pour y faire référence plus tard.

\begin{myrthm}{Théorème de Pythagore}{pythagore}
  Dans un triangle rectangle, le carré de la longueur de l'hypoténuse est égal à la somme des carrés des longueurs des deux autres côtés.
  \[ a^2 + b^2 = c^2 \]
\end{myrthm}

\textbf{Code source :}
\begin{texblock}
\begin{myrthm}{Théorème de Pythagore}{pythagore}
  Dans un triangle rectangle, le carré de la longueur de l'hypoténuse est égal à la somme des carrés des longueurs des deux autres côtés.
  \[ a^2 + b^2 = c^2 \]
\end{myrthm}
\end{texblock}

\subsection{Définitions (\texttt{myrdef})}

L'environnement \ci{myrdef} s'appuie sur la teinte verte (\texttt{infoGreen}) pour introduire de nouveaux concepts de manière distincte des théorèmes.

\begin{myrdef}{Espace Vectoriel}{espace_vect}
  Un espace vectoriel sur un corps $K$ est un ensemble muni d'une loi de composition interne (l'addition) et d'une loi de composition externe (la multiplication par un scalaire).
\end{myrdef}

\textbf{Code source :}
\begin{texblock}
\begin{myrdef}{Espace Vectoriel}{espace_vect}
  Un espace vectoriel sur un corps $K$ est un ensemble muni d'une loi de composition interne (l'addition) et d'une loi de composition externe (la multiplication par un scalaire).
\end{myrdef}
\end{texblock}

\subsection{Références croisées}

Le fonctionnement de ces boîtes intègre nativement la gestion des labels. Le deuxième argument que vous renseignez (par exemple \ci{pythagore}) génère automatiquement un identifiant interne préfixé par le type de boîte.

Pour faire référence à ces éléments dans votre texte, utilisez la commande standard \ci{\textbackslash ref\{...\}} accompagnée des préfixes \ci{thm:} ou \ci{def:} :

\begin{itemize}
    \item Pour citer le théorème précédent : \ci{\textbackslash ref\{thm:pythagore\}}
    \item Pour citer la définition précédente : \ci{\textbackslash ref\{def:espace\_vect\}}
\end{itemize}

\section{Blocs de Code}

La classe définit plusieurs environnements distincts, basés sur \texttt{tcolorbox} et \texttt{listings}, pour présenter du code source de manière élégante avec une barre latérale colorée.

\subsection{Code Python (\texttt{pyblock})}
\begin{pyblock}
def calculer_trajectoire(vitesse, angle):
    """Calcule la trajectoire en fonction de la vitesse et l'angle."""
    g = 9.81
    distance = (vitesse ** 2) * math.sin(2 * angle) / g
    return distance

# Exemple d'utilisation
print(f"Distance : {calculer_trajectoire(15, 45)}")
\end{pyblock}

\subsection{Ajouter de nouveaux langages}
La classe inclut une macro génératrice abstraite permettant d'ajouter très facilement de nouveaux langages pris en charge par le package \texttt{listings}. 

La syntaxe à utiliser dans votre préambule est la suivante :
\begin{texblock}
\newmyrtilleblock[Options]{nom_environnement}{Langage}{Couleur_barre}
\end{texblock}

\textbf{Exemple pour ajouter du JavaScript avec une bordure verte (\texttt{success}) :}
\begin{texblock}
\newmyrtilleblock{jsblock}{JavaScript}{success}
\end{texblock}

\textbf{Exemple pour du HTML sans numéros de ligne :}
\begin{texblock}
\newmyrtilleblock[numbers=none]{htmlblock}{HTML}{warn}
\end{texblock}

\subsection{Carte image (\texttt{imgbox})}

L'environnement \ci{imgbox} affiche une image dans une carte moderne avec ombre portée et coins arrondis. L'image occupe toute la largeur de la carte, suivie d'une bande de légende colorée avec barre latérale accent. Le titre est optionnel.

% ── PAGE DE GARDE ──────────────────────────────────
\section{Page de Garde}

La classe Myrtille intègre un générateur automatique de page de garde. La personnalisation s'effectue de manière claire et centralisée dans le préambule de votre document (avant le \ci{\textbackslash begin\{document\}}), à l'aide de macros dédiées.

\subsection{Variables de configuration}

Voici l'ensemble des commandes disponibles pour personnaliser les textes de votre couverture :

\begin{itemize}
    \item \ci{\textbackslash doctype\{...\}} : Le type de document (affiché en majuscules au-dessus du titre, ex: DOCUMENTATION, MANUEL).
    \item \ci{\textbackslash title\{...\}} : Le titre principal du document.
    \item \ci{\textbackslash subtitle\{...\}} : Le sous-titre ou la description courte.
    \item \ci{\textbackslash version\{...\}} : Le numéro de version affiché dans le badge en haut à droite (ex: v1.0). \textbf{Note :} Si cette commande est omise ou supprimée, le badge ne s'affichera tout simplement pas.
    \item \ci{\textbackslash coverfeatures\{...\}} : Un bloc libre, souvent utilisé pour insérer un petit tableau listant les points clés ou les chapitres principaux.
    \item \ci{\textbackslash coverfooter\{...\}} : Le texte discret affiché tout en bas de la page.
\end{itemize}

\subsection{Génération de la page}

Une fois les variables définies, il suffit d'appeler la commande \ci{\textbackslash makecover} juste après le début de votre document. Voici un exemple complet d'utilisation :

\begin{texblock}
% -- Dans le preambule de votre fichier main.tex --
\doctype{Manuel technique}
\title{Projet Open-Source}
\subtitle{Guide de déploiement et configuration}
\version{v2.4.1} % Supprimez cette ligne pour masquer le badge

\coverfeatures{
  \begin{tabular}{@{}ll@{}}
    \color{white!50}\sffamily\small Chapitre 1 & \color{accent}\sffamily\small \textbullet\ Installation \\[3pt]
    \color{white!50}\sffamily\small Chapitre 2 & \color{accent}\sffamily\small \textbullet\ Configuration \\
  \end{tabular}
}

\coverfooter{Généré automatiquement le \today}

% -- Apres le \begin{document} --
\begin{dοcument}

\makecover

% Suite de votre document...
\end{texblock}
